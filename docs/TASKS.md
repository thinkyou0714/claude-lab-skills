# TASKS — タスク状態・完了定義・引き継ぎサマリーの参照例

_進行中タスクの状態と完了定義（DoD）を記録する正本。_
_`success-failure-criteria` / `implementation-gate` / `cursor-handoff` / `summary-structuring` から_
_「現在のタスク状態」「完了定義」「引き継ぎサマリーの書式例」として参照される。_

---

## タスクの書式（テンプレート）

各タスクは以下の形式で記録する。完了条件は測定可能な形で書く。

```markdown
### T-<番号>: <タスク名>
- 状態: 未着手 / 進行中 / レビュー中 / 完了 / 保留
- 目的: 何のためにやるか（1文）
- 完了定義(DoD): どうなったら完了か（測定可能な形で、箇条書き）
- 失敗条件 / ロールバック発動基準: どうなったら戻すか
- 実行範囲: 触ってよいファイル・モジュール
- 影響範囲: 波及しうるシステム・機能
- 引き継ぎ先: 次に渡す担当（人 / 施工AI）
```

---

## 引き継ぎ用サマリーの参照例（summary-structuring 向け）

施工AI（Claude Code / Cursor）へ渡す際の最小サマリー例。

```markdown
## 引き継ぎサマリー: <タスク名>

- **What**: 何を実装するか（箇条書き）
- **DoD**: 完了条件（測定可能）
- **Scope**: 触ってよい範囲 / 触ってはいけない範囲
- **Rollback**: 失敗時の戻し方
- **Open Questions**: 未確定の前提（推測で埋めない）
```

## 完了定義（DoD）の一般原則

- 測定可能であること（「だいたい良くなった」は不可）
- 失敗条件とセットで定義すること
- 検証方法を明記すること（コマンド・確認手順）

詳細な観点は `success-failure-criteria` skill を参照。
