## Purpose

「自動化した方がいい気がする」を、数字に基づいた判断に変える。
手動・自動化それぞれの初期コスト・維持コスト・リスクコストを揃えて比較する。

## Use When

- automation-feasibility の後に、コスト比較を深掘りしたい場合
- 「どこまで自動化するか」の投資規模を決めたい場合
- 複数の自動化候補の優先度を決めたい場合
- 経営判断・承認申請のために ROI を示したい場合

## Inputs

以下を準備すること。不足している場合は推測せず、不足を明示する。

- **対象プロセス**: 比較するプロセスの説明
- **手動の現状コスト**: 頻度・所要時間・担当者コスト（時給換算可）
- **自動化の実装見積もり**: 初期実装工数の概算（不明な場合は「不明」と明示）
- **自動化の維持コスト**: 月次の監視・メンテナンス工数
- **比較期間**: 何ヶ月/年で比較するか

## Output Contract

以下の順で出力すること。順序を変えない。

1. **論点**: このコスト比較の核心（何が差を生むか）
2. **根拠**: その論点をそう判断した理由
3. **コスト比較表**: 定量比較（後述フォーマット）
4. **含意**: 数字が示す「本当に自動化すべき条件」
5. **改善案**: コストを下げる・ROI を改善する工夫
6. **代替案**: 部分自動化・ツール補助等の中間オプション
7. **判断材料**: 「今すぐ / 後回し / 手動継続」を選ぶための数字

### コスト比較表 フォーマット

```text
## コスト比較

### 手動継続
| 項目 | 値 | 備考 |
|---|---|---|
| 実行頻度 | 回/月 | |
| 1回あたり工数 | 分 | |
| 月次コスト | 時間/月 | |
| 年間コスト | 時間/年 | |
| エラー率・手戻りコスト | 概算 | |

### 自動化
| 項目 | 値 | 備考 |
|---|---|---|
| 初期実装工数 | 時間 | |
| 月次維持工数 | 時間/月 | |
| 外部サービスコスト | 円/月 | |
| 障害対応コスト（想定） | 時間/件 | |

### ROI 概算
| 指標 | 値 |
|---|---|
| 月次削減工数 | 時間/月 |
| 損益分岐点 | ヶ月後 |
| 1年後の累積削減 | 時間 |
```

## Review Lens

- **目的妥当性**: 比較している項目が、本来の判断に対して有意か
- **範囲の過不足**: 隠れたコスト（障害対応・監視・ドキュメント維持）を含んでいるか
- **中長期リスク**: 外部サービスの料金変更・廃止リスクを含んでいるか
- **LAB全体との整合性**: 他の自動化との重複コスト・共有コストを考慮しているか
- **非エンジニア理解可能性**: 「ヶ月後に元が取れる」という形で説明できるか
- **他LLM移植耐性**: 比較フレームが Claude 固有の形式に依存していないか

## Instructions

1. 手動コストを「頻度 × 1回工数」で月次換算する
2. 自動化コストを「初期実装 + 月次維持」に分解する
3. 損益分岐点を「初期コスト ÷ 月次削減工数」で算出する
4. 不明な数字は「不明: 確認が必要」として明示し、推測で埋めない
5. 隠れコスト（障害対応・監視・外部API料金）を別行で明示する
6. 「自動化すべき」の観点と「手動継続すべき」の観点を両方提示する
7. 最終判断は人間に委ねる

## Guardrails

- 数字を推測で埋めない。不明は「不明」として明示する
- 「自動化の方がコストが低い」と断定しない。隠れコストを含めた上で比較する
- ROI 回収期間が長い場合も、その数字を示す（隠さない）
- 比較期間（何ヶ月で比較するか）を明示する
- コスト比較の最終判断は人間が下す

## LAB Cross-Check

| 観点 | 状態 | 備考 |
|---|---|---|
| 自動化フロー | — | 既存フローとのコスト共有・重複を確認したか |
| データ / 認証 / ログ | — | ログ・監視のコストを含んでいるか |
| 実装 / 運用フロー | — | 障害対応コストを現実的に見積もったか |
| 非エンジニア理解可能性 | — | 「何ヶ月で元が取れる」という形で説明できるか |
| 会員共有 / 再利用耐性 | — | 比較フレームが他プロセスにも転用できるか |
| 他LLM移植耐性 | — | 計算フレームが Claude 固有に依存していないか |

状態は OK / 注意 / NG / 対象外 で記入すること。

## Handoff Notes

- **要件**: 自動化する場合の実装仕様
- **成功条件**: ROI が計画通りに推移している状態の定義
- **失敗条件**: コストが想定を超えた場合の判断基準
- **実行範囲**: 実装してよいファイル・ワークフロー・API
- **影響範囲**: コスト比較に関係する他の自動化・プロセス
- **ロールバック方針**: 自動化停止時の手動フォールバック手順
- **コスト比較**: 上記の比較表を転記
