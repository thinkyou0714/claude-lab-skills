## Purpose

「どちらも良さそう」または「どちらも悪そう」に見える選択肢を、軸を揃えて比較する。
最善の選択肢ではなく「この文脈における最適な妥協点」を見つける判断材料を提供する。

## Use When

- 選択肢が2つ以上あり、単純な優劣がつかない場合
- 速度 vs 品質、コスト vs リスク など複数軸が競合している場合
- decision-materials の前段として選択肢を深掘りしたい場合
- critique-panel 後に「それでも選ぶとしたら」を整理したい場合

## Inputs

以下を準備すること。不足している場合は推測せず、不足を明示する。

- **選択肢**: 比較する選択肢（最低2つ、5つ以内が望ましい）
- **比較軸の優先順位**: 何を最も重視するか（速度/コスト/品質/リスク/保守性等）
- **文脈**: なぜこの判断が必要か、どのフェーズか
- **制約**: 選択に影響する絶対条件（外せない制約）

## Output Contract

以下の順で出力すること。順序を変えない。

1. **論点**: このトレードオフの核心軸（何と何が競合しているか）
2. **根拠**: その論点をそう判断した理由
3. **トレードオフ比較表**: 多軸での比較（後述フォーマット）
4. **含意**: 比較が示す構造的な意味（どちらを選んでも生じる本質的な課題）
5. **改善案**: トレードオフを緩和・解消できる工夫（ハイブリッド案等）
6. **代替案**: 比較している選択肢以外の第3の道
7. **判断材料**: 優先軸に基づいて人間が選ぶための最終整理

### トレードオフ比較表 フォーマット

| 比較軸（優先順位順） | 選択肢A | 選択肢B | 備考 |
|---|---|---|---|
| （軸1: 最優先） | ◎/○/△/× | ◎/○/△/× | |
| （軸2） | | | |
| （軸3） | | | |
| ロールバック容易性 | | | |
| LAB整合性 | | | |

記号の意味: ◎ 優位 / ○ 許容 / △ 課題あり / × 不可

## Review Lens

- **目的妥当性**: 比較軸が本来の目的に対して適切か
- **範囲の過不足**: 重要な比較軸が抜けていないか
- **中長期リスク**: 短期評価と長期評価で逆転する軸がないか
- **LAB全体との整合性**: LMS / 自動化 / B2B 展開への影響を含んでいるか
- **非エンジニア理解可能性**: 記号と備考で非技術者に説明できるか
- **他LLM移植耐性**: 比較表の構造が Claude 固有の形式に依存していないか

## Instructions

1. 選択肢の名前と概要を1文で定義する
2. 比較軸を優先順位順に並べる（ユーザー指定がない場合は推定して明示）
3. 各セルを ◎/○/△/× で評価し、1行コメントを添える
4. トレードオフの「本質的な競合」を1〜2文で言語化する（これが論点）
5. トレードオフを緩和する工夫（段階的導入・ハイブリッド等）を提案する
6. 判断材料は「優先軸 X を重視するなら A、Y を重視するなら B」の形で整理する
7. 最終選択は人間に委ねる。AI は断定しない

## Guardrails

- どちらか一方を「正解」と断定しない
- 比較軸を後から増やして特定の選択肢に有利にしない
- 「どちらでも同じ」で終わらない。軸ごとの優劣を明示する
- コスト（工数・技術的負債・機会コスト）を省略しない
- ロールバック容易性を必ず含める

## LAB Cross-Check

| 観点 | 状態 | 備考 |
|---|---|---|
| 自動化フロー | — | 自動化への影響が比較軸に含まれているか |
| データ / 認証 / ログ | — | データ・認証設計への影響を確認したか |
| 実装 / 運用フロー | — | 運用コストが比較軸に入っているか |
| 非エンジニア理解可能性 | — | 記号＋備考で説明できているか |
| 会員共有 / 再利用耐性 | — | 比較表の構造が他ケースにも転用できるか |
| 他LLM移植耐性 | — | Claude 固有の評価基準に依存していないか |

状態は OK / 注意 / NG / 対象外 で記入すること。

## Handoff Notes

- **要件**: 選択した選択肢の仕様（トレードオフを了解した上で確定した内容）
- **成功条件**: 選択した軸での成功基準
- **失敗条件**: トレードオフのデメリットが顕在化した判断基準
- **実行範囲**: 実装対象のファイル・機能・DB
- **影響範囲**: 選択が波及するシステム・機能
- **ロールバック方針**: 選択が失敗した場合の戻し方
- **コスト比較**: 採用した選択肢のコスト見積もり
