# 用語集 — lab-core 共通基盤

_lab-skills 全体で使う用語の定義。スキル本文で用語がぶれたらここを正とする。_
_`search.py` の検索対象。新しい用語は表に追記する。_

---

## 中核概念

| 用語 | 定義 |
|------|------|
| 思考OS | 意思決定品質を高める論点整理・レビュー観点・判断材料の再利用可能な集合。本リポジトリの総称。 |
| Skill | 判断品質を上げる再利用単位（名詞句）。1 Skill = 1責務。`SKILL.md` で記述する。 |
| Command | 複数の Skill を束ねる薄い入口（動詞句）。AIツール固有のエントリーポイント。判断ロジックは持たない。 |
| Plugin | 領域別に Skill をまとめたパッケージ。`lab-<領域>` プレフィックスを持つ。 |
| SoT (Source of Truth) | 知識の正本。`src/` 配下に置く。AI固有フォルダ（`.claude/` 等）は SoT にしない。 |
| 判断材料 | 人間が次のアクションを選ぶために確認すべき情報。スキルは答えではなくこれを揃える。 |

## 判断・ゲート系

| 用語 | 定義 |
|------|------|
| 判断ゲート | 次フェーズへ進む前に答えるべき Yes/No の問いの集合。全て Yes になるまで進まない。詳細: [judgment-gates.md](../rules/judgment-gates.md) |
| 実装着手ゲート (GATE-1) | 実装に着手してよいかを確認するゲート。論点・成功条件・影響範囲・ロールバックを点検する。 |
| 自動化判断ゲート (GATE-2) | 手動を自動化してよいかを判断するゲート。頻度・失敗影響・手動境界を点検する。 |
| 公開・共有ゲート (GATE-3) | 外部公開・会員共有前の点検。個人名・内部パス・私的情報の混入を確認する。 |
| 費用対効果ゲート (GATE-4) | API/SaaS 導入前の点検。月次費用と工数削減効果を比較する。 |
| 手動境界 | 自動化せず人間が必ず判断する操作。失敗影響が大きい箇所に残す。 |

## 実装・自動化系

| 用語 | 定義 |
|------|------|
| 影響範囲 (blast radius) | ある変更が波及しうる範囲。直接・依存・副作用の3分類で洗い出す。 |
| ロールバック | 失敗時に変更前の状態へ戻す操作。発動基準（失敗条件）を事前に定義する。 |
| 冪等性 (idempotency) | 同じ操作を複数回実行しても結果が一度の実行と同じになる性質。リトライ設計の前提。 |
| トリガー / アクション | 自動化フローの起点（トリガー）と実行内容（アクション）。trigger-action-map で整理する。 |
| 障害点 (failure point) | フローが失敗しうる箇所。failure-point-review で特定する。 |
| SPOF（単一障害点） | そこが停止すると全体が停止する箇所（Single Point of Failure）。最優先で冗長化・検知を検討する。 |
| カスケード障害 | 一部の障害が連鎖して全体へ波及する障害。手動境界・サーキットブレーカで断つ。 |
| サイレント障害 | エラーにならずに失敗している状態。検知（監視・整合性チェック）が無いと気づけない。 |
| 施工AI | 実装・ファイル操作を担う AI（Claude Code / Cursor / Codex CLI）。 |
| handoff | 設計から施工AIへ実行可能な指示書を渡すこと。cursor-handoff skill が生成する。 |

## 事業・運用系

| 用語 | 定義 |
|------|------|
| manual-first | 最初は手動で運用し、ボトルネックを観察してから自動化を検討する方針。 |
| ROI | 投資対効果。自動化では「削減工数 × 時給換算 − 月額費用」で簡易試算する。 |
| ロックイン | 特定 SaaS/ツールへの依存度。撤退・移行コストの高さで評価する。 |
| PII | 個人を特定しうる情報（Personally Identifiable Information）。公開・ログ出力時に要注意。 |
| ステークホルダー | 施策の影響を受ける人・チーム・ユーザー。非エンジニアを含む。 |
| 主力プロダクト（LMS） | 中核プロダクト。学習管理（LMS）を中心とする。スキル本文で事業文脈の基準として参照される（公開版では個別の製品名・内部フェーズ名は記載しない）。 |
| B2B 展開 | 法人向け提供・販売の展開方針。LMS・自動化と並ぶ事業の柱。 |
