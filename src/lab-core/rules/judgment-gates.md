# 判断ゲート — lab-core 共通基盤

_フェーズ移行・設計確定・実装着手前に答えるべき問いの集合。_
_すべてに Yes と言える状態になるまで次へ進まない。_

---

## GATE-1: 実装着手ゲート

着手前に以下を確認すること。

| # | 問い | 確認方法 |
|---|------|----------|
| G1-1 | 論点は1文で言えるか | issue-framing を通す |
| G1-2 | 成功条件が測定可能か | success-failure-criteria を確認 |
| G1-3 | 影響範囲（blast radius）を言えるか | boundary-check を通す |
| G1-4 | ロールバック手順があるか | 実装計画に明記 |
| G1-5 | 既存の本番コードを壊さないか | 侵襲的変更に該当しないか確認 |

## GATE-2: 自動化判断ゲート

手動 → 自動化の検討時に使う。

| # | 問い | 判断基準 |
|---|------|----------|
| G2-1 | 月 10 回以上繰り返すか | Yes → 自動化候補 |
| G2-2 | 失敗時の影響は小さいか | No（失敗影響大）→ 手動維持 |
| G2-3 | 再現手順が文書化されているか | No → 自動化前に文書化を先行 |
| G2-4 | 人間の判断が必要なステップがあるか | Yes → そこは手動境界として残す |
| G2-5 | 自動化コストが工数削減効果を下回るか | Yes → 手動維持または半自動を選ぶ |

## GATE-3: 公開・共有ゲート

外部公開・会員共有前に確認。

| # | 問い | 確認方法 |
|---|------|----------|
| G3-1 | 個人名・内部パスが含まれないか | grep で確認 |
| G3-2 | WORK_PRIVATE の情報が混入していないか | ディレクトリ分離を確認 |
| G3-3 | 外部公開想定のリポジトリか | LAB_PUBLIC のみ可 |

## GATE-4: 費用対効果ゲート

API/SaaS 導入前に確認。

| # | 問い | 判断基準 |
|---|------|----------|
| G4-1 | 月額費用は工数削減効果を下回るか | 試算を出してから判断 |
| G4-2 | 無料枠またはローカル代替があるか | 先に調査する |
| G4-3 | 解約・移行コストが許容範囲か | ロックイン度を確認 |
