# PII 機微度の分類 — lab-data-auth

_個人情報（PII）の機微度を分類する参照表。`pii-handling-review` / `audit-log-design` が使う。_

---

## 機微度の分類

| 機微度 | 例 | 取り扱い方針 |
|---|---|---|
| 高 | 決済情報・パスワード・認証トークン・健康情報 | 原則保持しない。必要なら暗号化＋アクセス厳格化。ログ出力禁止 |
| 中 | 氏名・メール・電話・住所 | 最小収集。マスキング。アクセス制限。保持期間を定める |
| 低 | 表示名・地域（粗い粒度）・利用統計（匿名） | 目的内で利用。匿名化を優先 |

---

## ログ・通知に出してはいけない項目

- パスワード・トークン・APIキー・秘密鍵
- 決済情報（カード番号等）
- 高機微の個人情報

> 代わりに識別子（ユーザーID）や相関 ID を使い、PII 本体はログに残さない。

---

## 保持期間の考え方

- 目的が終わったら削除する（無期限保持を既定にしない）
- 削除手段（誰が・いつ・どう消すか）を設計時に決める
- 開示・削除請求に応えられる構造にする
